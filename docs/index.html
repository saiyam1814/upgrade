Why another upgrade tool

Every existing tool — pluto, kubent, every cloud's upgrade insights — checks only what's in etcd plus a recent audit log. None inspect the workload supply chain (Git, Helm, ArgoCD/Flux, third-party CRDs, admission webhooks, addon controllers, PV/PVCs). And once the upgrade starts, you are on your own.

🛡️

Read-only by default

Every command is safe to run in production. Mutating actions require --execute + per-action [y/N].

🧭

Conductor, not orchestra

Never runs cloud CLIs for you. Emits the exact aws eks / gcloud / az command — you run it.

🔁

Watch + unstick

Detects stuck PDBs, stalled CRD migrations, addon dep cycles, NotReady nodes — emits the unstick command.

💾

PV/PVC aware

Catches in-tree → CSI migration risks, single-replica StatefulSets, RWX volumes on cloud classes mid-removal.

🌐

vCluster fleet

Bumping the host? fleet --host-target v1.34 checks every tenant's vCluster version against a host-K8s compat matrix and tells you which tenants must bump first.

⚡

Single static binary

One Go binary. krew · brew · curl · go install. No daemons, no telemetry, no network egress.

Install

Pick whichever fits. All paths give you the same single static binary.

kubectl krew install upgrade
kubectl upgrade --help
brew install saiyam1814/tap/kubectl-upgrade
kubectl upgrade --help
curl -sSL https://raw.githubusercontent.com/saiyam1814/upgrade/main/install.sh | sh
kubectl-upgrade --help
go install github.com/saiyam1814/upgrade@latest
# binary lands as $(go env GOBIN)/upgrade — rename to kubectl-upgrade for kubectl plugin invocation

Fleet — host × vCluster compat

When you bump a Control Plane Cluster's K8s version, not every vCluster on it tolerates the new host. fleet checks every tenant against a curated compat matrix and tells you which tenants must bump first.

$ kubectl upgrade fleet --host-target v1.34 --plan
vCluster bandSupported host K8s window
v0.20 – v0.211.26 … 1.30
v0.22 – v0.231.27 … 1.31
v0.241.28 … 1.32
v0.25 – v0.271.29 … 1.33
v0.28 – v0.301.29 … 1.34
v0.31 – v0.331.30 … 1.35
v0.34+1.31 … 1.36

Per tenant: ✗ BLOCKER if outside the window (run-book emits the minimum vCluster version to bump to first), ⚠ MEDIUM at the upper edge, ✓ INFO if safe. Hand-curated; PRs welcome to extend.
